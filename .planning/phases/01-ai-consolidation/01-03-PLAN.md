**Verify model string:**

**Fix retry exception handling in `call_claude_analyze()`:**
Current code catches `anthropic.RateLimitError` with `time.sleep(30)` but does NOT break out of the loop properly — it just sleeps and falls through to the next attempt. Add `continue` after `time.sleep()` in the RateLimitError handler to correctly retry.

More importantly, add `anthropic.OverloadedError` to the retry logic. The current code only catches `anthropic.RateLimitError` (HTTP 429) but misses `anthropic.OverloadedError` (HTTP 529, Anthropic-specific overload). Add it to the same except clause:

```python
except (anthropic.RateLimitError, anthropic.OverloadedError):
    if attempt == max_retries - 1:
        return {"error": "rate_limit_or_overloaded"}
    sleep_time = (2 ** attempt) * 10  # 10s, 20s, 40s
    time.sleep(sleep_time)
    continue
```

Also add `anthropic.APIConnectionError` and `anthropic.APITimeoutError` handling (currently both fall into generic `except Exception`):
```python
except (anthropic.APIConnectionError, anthropic.APITimeoutError):
    if attempt == max_retries - 1:
        return {"error": "connection_or_timeout"}
    time.sleep(2 ** attempt)
    continue
```

The generic `except Exception` can stay as the final fallback for unexpected errors.

**Add cost tracking to `call_claude_analyze()`:**
On successful API response, before returning the parsed JSON, extract and include token usage:
```python
parsed["input_tokens"] = response.usage.input_tokens
parsed["output_tokens"] = response.usage.output_tokens
```
Return the enriched dict (the caller `analyze_defect()` will extract cost from it).

Assign to the `DeepAnalysis` object: `cost_usd=cost_usd`. On error path, `cost_usd=0.0`.
