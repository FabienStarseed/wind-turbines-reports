  <name>Task 1: Create backend/database.py — SQLAlchemy engine, Job model, CRUD helpers</name>
  <files>backend/database.py</files>
  <action>
Create backend/database.py as a new file. This is the complete persistence layer. Implement exactly as specified in 02-RESEARCH.md Pattern 1 + Pattern 3 + the complete database.py code example. Key requirements:

DATA_DIR setup (at module import time, before engine creation — per Pitfall 6):
- `DATA_DIR = Path(os.environ.get("DATA_DIR", "/data"))`
- `DATA_DIR.mkdir(parents=True, exist_ok=True)` — must run at import time
- `JOBS_DIR = DATA_DIR / "jobs"`
- `JOBS_DIR.mkdir(parents=True, exist_ok=True)`
- `DB_PATH = DATA_DIR / "bdda.db"`
- `DATABASE_URL = f"sqlite:///{DB_PATH}"`

Engine (per Pitfall 2 — check_same_thread=False is mandatory):
- `create_engine(DATABASE_URL, connect_args={"check_same_thread": False})`
- WAL mode via `@event.listens_for(engine, "connect")` — executes `PRAGMA journal_mode=WAL` and `PRAGMA synchronous=NORMAL` (per Pitfall 3 — WAL prevents status polling from blocking)

Session factory:
- `SessionLocal = sessionmaker(bind=engine, autocommit=False, autoflush=False)`
- `get_db()` as a `@contextmanager` that yields `SessionLocal()` and closes in finally block (does NOT auto-commit — per Pitfall 4)

Job model (DeclarativeBase, NOT legacy declarative_base() — per State of the Art):
- `class Base(DeclarativeBase): pass`
- `class Job(Base)` with `__tablename__ = "jobs"` and these columns:
  - `id`: Integer primary key, autoincrement
  - `job_id`: String(8), unique, not null, indexed
  - `stage`: String(32), not null, default="queued"
  - `stage_message`: Text, default=""
  - `turbine_meta_json`: Text, default="{}" — stores dict as JSON string (per Pitfall 5 — use Text not JSON type)
  - `image_count`: Integer, default=0
  - `total_images`: Integer, nullable
  - `flagged_images`: Integer, nullable
  - `critical_findings`: Integer, nullable
  - `pdf_path`: Text, nullable
  - `triage_cost_usd`: Text, nullable — stored as string (per Pitfall 5 / Anti-Pattern: float precision)
  - `classify_cost_usd`: Text, nullable
  - `analyze_cost_usd`: Text, nullable
  - `total_cost_usd`: Text, nullable
  - `image_cap_warning`: Text, nullable
  - `error_traceback`: Text, nullable
  - `created_at`: DateTime(timezone=True), server_default=func.now()
  - `updated_at`: DateTime(timezone=True), onupdate=func.now()
  - `completed_at`: DateTime(timezone=True), nullable
  - `failed_at`: DateTime(timezone=True), nullable

`init_db()` function:
- `Base.metadata.create_all(bind=engine)` — idempotent, safe to call on every startup

CRUD helpers (use SQLAlchemy 2.0 select() style, NOT legacy session.query() — per State of the Art):
- `_job_to_dict(job: Job) -> dict` — converts ORM row to dict. Use `json.loads(job.turbine_meta_json or "{}")` for turbine_meta. Convert cost Text fields back to float with `float(v) if v else None`. Convert datetime fields to `.isoformat()` if not None.
- `get_job(job_id: str) -> Optional[dict]` — `select(Job).where(Job.job_id == job_id)` via `session.scalars().first()`
- `save_new_job(state: dict)` — inserts new Job row; `turbine_meta_json = json.dumps(state.get("turbine_meta", {}))`; commits
- `update_job(job_id: str, **kwargs)` — maps kwargs to column names; converts cost floats to str for Text columns; uses `update(Job).where(Job.job_id == job_id).values(**values)`; commits
- `set_stage(job_id: str, stage: str, message: str = "")` — calls `update_job(job_id, stage=stage, stage_message=message)`
- `list_jobs_last_30_days() -> list[dict]` — queries jobs with `created_at >= datetime.now(timezone.utc) - timedelta(days=30)`, orders by `created_at.desc()`, returns list of dicts (job_id, stage, created_at, completed_at, turbine_meta parsed from JSON)

Use `select()` + `session.scalars()` throughout (2.0 style). Import `select, update` from `sqlalchemy`.
