```yaml
# IMPORTANT: Persistent disk requires a paid web service instance.
# Minimum: Starter tier ($7/month). Free tier has ephemeral filesystem — data
# is wiped on every restart/redeploy. Upgrade the web service to Starter or
# higher before deploying with this disk config.
# Disk cost: $0.25/GB/month ($0.25/month for 1GB).
# Total cost with Starter tier: ~$7.25/month.
# Zero-downtime deploys are NOT available when a persistent disk is attached
# (Render limitation). Brief downtime during redeploy is expected and acceptable.

services:
  - type: web
    name: bdda
    runtime: python
    rootDir: .
    buildCommand: pip install -r requirements.txt
    startCommand: cd backend && uvicorn api:app --host 0.0.0.0 --port $PORT
    healthCheckPath: /api/health
    disk:
      name: bdda-data
      mountPath: /data
      sizeGB: 1
    envVars:
      - key: ANTHROPIC_API_KEY
        sync: false
      - key: COST_LIMIT_USD
        sync: false
```

Key changes from existing render.yaml:

- Removed `GOOGLE_API_KEY` (removed from pipeline in Phase 1)
- Removed `KIMI_API_KEY` (removed from pipeline in Phase 1)
- Added prominent comment block explaining paid-tier requirement

NOTE: `DATA_DIR` env var does NOT need to be listed — the default in database.py is `/data` which matches the mountPath exactly. Only add DATA_DIR to render.yaml if a non-standard path is needed.

The mountPath `/data` in render.yaml MUST match the default `DATA_DIR = Path(os.environ.get("DATA_DIR", "/data"))` in database.py. Both use `/data`.
