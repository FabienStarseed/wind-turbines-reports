3. **BDDAReport class** — subclass of FPDF:
   - `__init__(self, report_data: dict)`: Call super with A4 portrait, mm units. Store report_data and report_ref. Call `_register_fonts()`. Set margins (left=15, top=20, right=15). Set auto page break with 20mm margin.
   - `_register_fonts(self)`: Add Inter Regular, Bold, Italic from FONT_DIR. If files don't exist, fall back to Helvetica with a warning print.
   - `header(self)`: Skip on page 1 (cover). Otherwise: brand name "DroneWind Asia" left-aligned bold 9pt in BRAND_NAVY, report ref right-aligned 8pt in grey, thin line underneath, 4mm gap. If a logo PNG exists at configurable path, use `self.image()` instead of text (but default to text).
   - `footer(self)`: Skip on page 1 (cover). Otherwise: "Page N | Confidential — DroneWind Asia Inspection Report" centered 7pt grey, positioned at -12mm.

4. **Helper: `hex_to_rgb(hex_color: str) -> tuple`** — convert CSS hex to RGB tuple.

5. **`_build_page_map(report_data: dict) -> dict`** — Pre-calculate page numbers for TOC. Cover=1, TOC=2, Exec Summary=3. Then for each blade: 1 page per defect (or 1 page if no defects). Then action matrix (1 page). Then 1 blade map per blade. Then inspection details. Return dict mapping section names to page numbers.

6. **`_render_cover(pdf, report_data)`** — Full-page cover:
   - Large "DroneWind Asia" text or logo centered at top
   - "Wind Turbine Blade Inspection Report" title, large bold
   - Turbine ID, site name, country in medium text
   - Inspection date, inspector name
   - Report reference number
   - Condition rating badge: coloured rounded rect with condition letter + label (use CONDITION_RATINGS colours converted to RGB)
   - "CONFIDENTIAL" footer text

7. **`_render_toc(pdf, report_data, page_map)`** — Table of contents on page 2:
   - "Table of Contents" heading
   - Dotted leader lines from section name to page number
   - Sections: Executive Summary, each blade's findings, Action Matrix, each blade's map, Inspection Details

8. **`_render_executive_summary(pdf, report_data)`** — Page 3:
   - "Executive Summary" heading
   - Overall condition rating with large coloured badge and description text
   - Triage statistics: total images, flagged, clean, flag rate (if triage_stats available)
   - Defect summary table: Cat 0-4 rows with count and colour band (use SEVERITY_COLORS_IEC)
   - Total defects, critical count
   - Top 3-5 critical findings (brief: defect_name, blade, zone, category)
   - Recommendation paragraph based on condition rating

9. **`_render_inspection_details(pdf, report_data)`** — Final page:
   - "Inspection Details" heading
   - Two-column layout with label:value pairs:
     - Turbine Model, Hub Height, Rotor Diameter, Blade Length
     - Drone Model, Camera
     - Weather, Wind Speed, Temperature, Visibility
     - GPS coordinates
     - Inspector Notes
   - Generated at timestamp
   - Legal disclaimer: "This report was generated using AI-assisted defect detection..."
   - Engineer review count if > 0

10. **`generate_pdf_fpdf2(report_data: dict, output_path: Path) -> Path`** — Main entry point:
    - Create BDDAReport instance
    - Build page_map
    - Call _render_cover, _render_toc, _render_executive_summary
    - (Defect pages and blade maps will be added by Plan 03 — leave commented placeholder)
    - Call _render_inspection_details
    - pdf.output(str(output_path))
    - Return output_path

11. **Update `build_report()`** — Change to call `generate_pdf_fpdf2()` instead of `render_html()` + `generate_pdf()`. Remove references to templates_dir, css_path, save_html. Keep the data loading and build_report_data() call.
