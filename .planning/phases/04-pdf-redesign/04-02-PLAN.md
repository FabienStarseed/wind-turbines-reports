In the `run_pipeline()` function in api.py, after triage completes and before the `images_dir` deletion, add thumbnail saving logic:

1. **After triage results are saved (line ~238) and before the `shutil.rmtree(images_dir)` call (line ~243):**

   ```python
   # Save thumbnail copies of flagged images before deleting originals
   # These survive for classify stage and PDF report embedding (PDF-03)
   thumbnails_dir = job_dir / "thumbnails"
   thumbnails_dir.mkdir(exist_ok=True)
   flagged_paths = {str(r.image_path) for r in summary.results if r.has_defect}
   for orig_path_str in flagged_paths:
       orig_path = Path(orig_path_str)
       if orig_path.exists():
           thumb_path = thumbnails_dir / orig_path.name
           try:
               with Image.open(orig_path) as img:
                   img.thumbnail((1568, 1568), Image.LANCZOS)
                   img.convert("RGB").save(str(thumb_path), "JPEG", quality=85)
           except Exception:
               # If thumbnail fails, copy original (better than nothing)
               import shutil as _shutil
               _shutil.copy2(str(orig_path), str(thumb_path))
   ```

   Add `from PIL import Image` at the top of api.py if not already imported. The `Image` import is already available since Pillow is in requirements.txt.

2. **Update the `flagged` list construction (line ~250-253):**

   After building the flagged list, update each entry's `path` to point to the thumbnail copy:

   ```python
   # Update flagged image paths to point to thumbnail copies
   for f in flagged:
       orig_name = Path(f["path"]).name
       thumb = thumbnails_dir / orig_name
       if thumb.exists():
           f["path"] = str(thumb)
   ```

   This ensures classify_batch reads from the thumbnail files which still exist after originals are deleted.

3. **Important:** The `image_path` stored in classify_data (and ultimately in the PDF report's `report_data["blade_findings"]`) will now point to `thumbnails/` paths. This means the PDF can embed these images.

4. **Keep the existing `shutil.rmtree(images_dir)` call unchanged** — the originals (15-30MB each) are still deleted. Only the thumbnails (~50-200KB each at 1568px) survive.

5. **Note on disk impact:** With 80 flagged images max at ~150KB each = ~12MB total for thumbnails. Well within the 1GB persistent disk budget.
