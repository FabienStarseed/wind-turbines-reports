# Plan 05-04 — Job History Table + style.css + app.js Cleanup

## Goal

1. Populate the HISTORY tab in `index.html` with the Cyber Operator data-dense job table
2. Rewrite `frontend/style.css` with all Cyber Operator base styles, CSS variables, keyframes, and utility classes that Tailwind CDN cannot produce (`action-btn`, `cyber-input`, `scanlines`, animated background, `pulse`)
3. Strip `frontend/app.js` of all logic (now in Alpine), leaving only a no-op comment file to avoid 404

## must_haves

- History table has 7 columns: DATE / TURBINE ID / SITE / STATUS / DEFECTS / COST / DOWNLOAD
- `loadHistory()` calls `apiFetch('/api/jobs')` — Bearer token required (fixes existing 401 bug)
- History only loads on first HISTORY tab click (lazy load), not on page load
- Download button uses same `downloadReport(jobId)` blob pattern — greyed out (disabled) for non-complete jobs
- RUNNING status badge has amber `pulse` class applied reactively
- Empty state: "NO REPORTS GENERATED YET" centered muted text
- `style.css` defines all CSS variables (`:root`), body `::before`/`::after` animated background, `scanlines` div, `action-btn` clip-path, `cyber-input`, `pulse` keyframe, `[x-cloak]` hide rule
- `app.js` is not deleted (404 would surface) but contains only a comment — all logic lives in inline `<script>` in `index.html`
- No white/light backgrounds anywhere
- No border-radius > 4px anywhere in CSS

    Also implement `loadHistory()` method in the `app()` inline script:
    ```js
    async loadHistory() {
      try {
        const res = await this.apiFetch('/api/jobs');
        if (!res || !res.ok) return;
        this.jobs = await res.json();
        this.historyLoaded = true;
      } catch {
        this.historyLoaded = true; // Don't leave it in loading state
      }
    },
    ```
