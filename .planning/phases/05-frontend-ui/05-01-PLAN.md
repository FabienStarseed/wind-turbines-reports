# Plan 05-01 — Login Page Rewrite

## Goal

Full rewrite of `frontend/login.html` with Cyber Operator design system, Alpine.js `loginApp()` component, and Tailwind CDN. Must authenticate correctly against FastAPI OAuth2 endpoint using `URLSearchParams`.

## must_haves

- Login page renders with dark `#050510` background, animated radial gradient + grid scanlines
- AWID branding: "AWID" in Bebas Neue cyan gradient, subtitle "APAC WIND INSPECTIONS DRONES" in muted mono
- Form submits with `URLSearchParams` + `Content-Type: application/x-www-form-urlencoded` (FastAPI OAuth2 requirement)
- JWT stored in `localStorage['bdda_token']`, username decoded from JWT payload stored in `localStorage['bdda_username']`
- Error message displayed in magenta (`#FF006E`) on failed login
- Loading state: button text changes to "AUTHENTICATING…", disabled during request
- Already-logged-in users are redirected to `/` or `?next=` param immediately
- `action-btn` clip-path style applied to submit button (cyan bg, dark text, polygon clip)
- No border-radius > 4px anywhere
- Page is readable and usable on tablet (768px+) — form centered, max-width constrained

  <task id="05-01-T2" name="Write loginApp() Alpine component with auth logic">
    Add inline `<script>` before `</body>` containing `loginApp()` function:

    ```js
    function loginApp() {
      return {
        username: '', password: '', loading: false, errorMsg: '',

        checkAlreadyLoggedIn() {
          if (localStorage.getItem('bdda_token')) {
            const params = new URLSearchParams(window.location.search);
            window.location.href = params.get('next') || '/';
          }
        },

        async login() {
          this.loading = true;
          this.errorMsg = '';
          try {
            const res = await fetch('/api/auth/token', {
              method: 'POST',
              body: new URLSearchParams({ username: this.username, password: this.password }),
              headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
            });
            if (!res.ok) {
              this.errorMsg = 'Invalid credentials. Please try again.';
              return;
            }
            const data = await res.json();
            localStorage.setItem('bdda_token', data.access_token);
            // Decode username from JWT payload
            try {
              const payload = JSON.parse(atob(data.access_token.split('.')[1]));
              localStorage.setItem('bdda_username', payload.sub);
            } catch {}
            const params = new URLSearchParams(window.location.search);
            window.location.href = params.get('next') || '/';
          } catch {
            this.errorMsg = 'Network error. Please check connection.';
          } finally {
            this.loading = false;
          }
        },
      }
    }
    ```

    Form element: `<form @submit.prevent="login()">` — no id needed, no vanilla JS event listeners.

    Button markup:
    ```html
    <button type="submit" class="action-btn w-full justify-center mt-4"
            :disabled="loading">
      <span x-show="!loading">▶ LOGIN</span>
      <span x-show="loading">AUTHENTICATING…</span>
    </button>
    ```

    Inputs must use `x-model`:
    - `<input x-model="username" type="text" name="username" class="cyber-input w-full" autocomplete="username" required autofocus>`
    - `<input x-model="password" type="password" name="password" class="cyber-input w-full" autocomplete="current-password" required>`
